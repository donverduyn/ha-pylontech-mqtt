# Single source of truth for every AI CLI config/auth file that needs to
# survive a container rebuild, read by three scripts (host-side
# seedHostAgentConfig.sh, and container-side postCreate.sh + syncAgentConfigOut.sh)
# so the list of files is never duplicated. Format: "<path relative to
# $HOME>|<seed kind>", where seed kind is:
#   json  - missing file is created as an empty JSON object: {}
#   empty - missing file is created zero-length (YAML/TOML/plain-token files,
#           where an empty file parses the same as "no config")
# GitHub Copilot CLI's ~/.copilot is deliberately not here: it's a whole
# directory (session-store.db + friends), mounted directly rather than
# staged through this file-by-file sync.

# Claude Code
